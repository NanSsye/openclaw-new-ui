## 一、聊天页面 (Chat) 优化

### 1.1 顶部控制栏布局优化
**问题**: 按钮横向排列时容易误触，且占用空间过大

**解决方案**:
- 所有控制按钮改为图标模式，节省屏幕空间
- 按钮顺序调整为：**模型选择(左) → 会话选择(左) → 命令控制台 → 用量统计 → 详情开关(右)**
- 字体不再加粗，避免显示不全

### 1.2 模型选择器增强
**问题**: 选择模型后无法立即生效

**解决方案**:
- 选择模型时自动发送 `/model <模型ID>` 命令到网关
- 下拉菜单显示模型所属 Provider 分组
- 每个 Provider 使用不同颜色标识

### 1.3 用量统计下拉化
**问题**: 用量统计使用弹窗模式，切换麻烦

**解决方案**:
- 改为图标下拉菜单，与命令控制台风格统一
- 打开时自动加载最新用量数据
- 显示加载动画

### 1.4 会话选择器优化
**问题**: 会话列表无法快速区分不同 Agent

**解决方案**:
- 按 Agent 分组显示会话列表
- 每个 Agent 使用唯一颜色标识（基于 Agent ID 哈希）
- 当前会话高亮显示

### 1.5 AI 消息文本溢出修复
**问题**: AI 回复内容过长时导致页面可左右滚动

**解决方案**:
- 消息内容区域 `overflow-x: hidden`
- 聊天消息最大宽度限制

## 二、代理页面 (Agents) 移动端适配

### 2.1 响应式布局
**问题**: 左侧代理列表在小屏幕上无法正常显示

**解决方案**:
- **桌面端 (md+)**: 左侧显示代理列表卡片
- **移动端**: 隐藏左侧列表，顶部显示下拉选择器

### 2.2 下拉选择器样式统一
**问题**: 原生 select 样式与整体 UI 不协调

**解决方案**:
- 使用与聊天页一致的下拉菜单样式
- 显示当前选中的代理名称
- 点击展开代理列表

### 2.3 Tab 标签优化
**问题**: Tab 标签在小屏幕上换行或溢出

**解决方案**:
- Tab 标签简化文字（概览/配置/工具/技能）
- 支持横向滚动
- 移动端缩小间距

## 三、配置页面 (Config) 移动端适配

### 3.1 左侧导航响应式
**问题**: 左侧配置分类在小屏幕上无法正常显示

**解决方案**:
- **桌面端 (sm+)**: 左侧显示配置分类卡片
- **移动端**: 隐藏左侧列表，顶部显示下拉选择器

### 3.2 下拉选择器样式统一
**解决方案**:
- 使用与聊天页一致的下拉菜单样式
- 显示当前选中的配置分类
- 包含分类图标

### 3.3 安全区域适配
**解决方案**:
- 添加 `env(safe-area-inset-top)` 和 `env(safe-area-inset-bottom)` 适配
- 确保内容不被系统状态栏和导航栏遮挡

## 四、全局安全区适配

### 4.1 状态栏适配
**问题**: Web 内容与安卓状态栏重叠，无法点击

**解决方案**:
- 修改 Android 原生 `MainActivity.java`
- 添加 `WindowInsetsCompat` 处理
- 配置 `viewport-fit=cover` 和安全区 CSS

### 4.2 CSS 安全区
**解决方案**:
- `(app)/layout.tsx`: 添加 `padding-top: env(safe-area-inset-top)`
- `(auth)/layout.tsx`: 同样添加安全区适配
- 全局 `globals.css`: 设置安全区变量

### 8.1 性能优化
- `useCallback` 优化事件处理函数

- 使用 `cn()` 工具函数合并类名

- 响应式设计：`sm:` `md:` `lg:` 前缀
- 安全区适配：`env(safe-area-inset-*)`
- 触控优化：增大点击区域
